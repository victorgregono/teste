## O que é este diretório
Este projeto usa **Spec-Driven Development (SDD)** para transformar ideias em implementação com mais contexto, clareza e rastreabilidade.

## Fluxo padrão
1. Criar PRD em `docs/prd/`
2. Criar plano técnico em `docs/architecture/`
3. Criar task breakdown em `docs/tasks/`
4. Implementar por task
5. Testar e validar
6. Registrar decisões importantes em `docs/decisions/`

## Regra prática
Não iniciar uma feature relevante direto pelo código.

## Objetivo
- dar mais contexto para a IA
- reduzir retrabalho
- melhorar qualidade técnica
- aproximar a execução do raciocínio do Scrum
